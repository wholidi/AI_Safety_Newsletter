# 🧩 System Architecture

This system is designed as a **layered AI governance pipeline**.

---

## Layers

### 1. Input Layer
- RSS ingestion
- preprocessing & cleaning

---

### 2. Agent Layer
- Agent 0 → selection
- Agent 1 → generation
- Agent 2 → evaluation
- Agent 3 → revision

---

### 3. Guardrail Layer
- schema validation
- quality enforcement
- safety checks

---

### 4. Policy Layer
- internal governance rules
- compliance constraints
- disclosure requirements

---

### 5. Verification Layer
- risk scoring
- severity classification
- control mapping
- audit evidence

---

### 6. Decision Layer
- approve / revise / reject routing

---

### 7. Output Layer
- HTML review pack
- human decision interface

---

## Key Idea

This architecture ensures:

> AI output is continuously evaluated, constrained, and transformed into evidence

---

## Strategic Positioning

This system acts as:

> **AI Governance Runtime Layer (Verification + Control + Audit)**
